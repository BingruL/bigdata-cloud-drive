# 构建一个基于 HBase/HDFS 的智能云盘系统

## 项目背景

这是大数据技术基础课程的期末大作业。老师的原话是「推荐做 HBase 网盘以及关于 Token 相关的项目，包含存储、计算，结合 AI。要求界面精美，功能健全完善」。我需要先做出完整可运行的系统，学期末进行项目演示。

## 推荐技术栈

- 后端用 Python Flask
- 文件内容存储用 Hadoop HDFS（通过 `hdfs` 库的 WebHDFS 接口）
- 文件元数据、用户信息、操作日志等结构化数据存储用 Apache HBase（通过 `happybase` 库连接 Thrift Server）
- 用户认证用 JWT（PyJWT）
- 分布式计算用 PySpark（写独立的 Spark 作业脚本，用 spark-submit 运行）
- AI 功能调用兼容 OpenAI 格式的 LLM API（如本地 Ollama），用于文件摘要和标签生成
- 前端用 Vue 3 + ECharts，通过 CDN 引入

## 需要实现的功能

### 1. 用户认证与权限控制

- 用户注册、登录
- 登录成功后返回 JWT Token，后续所有接口都通过 Token 校验身份
- Token 过期机制与刷新
- 密码加密存储
- 两种角色：普通用户和管理员。普通用户只能操作自己的文件，管理员可以查看所有文件和日志

### 2. 文件管理

- **上传**：用户上传文件后，文件内容存入 HDFS，文件元数据（文件名、大小、类型、所属用户、HDFS 路径、上传时间、下载次数等）存入 HBase
- **文件列表**：用户登录后看到自己的文件列表，支持按文件名搜索、按类型筛选、分页
- **下载**：先从 HBase 查到文件的 HDFS 路径，再从 HDFS 读取文件返回给用户。同时增加下载计数
- **删除**：同步删除 HDFS 中的文件和 HBase 中的元数据
- **搜索**：支持按文件名关键字、文件类型、上传时间范围组合查询

### 3. 操作日志

记录所有用户行为（登录、注册、上传、下载、删除），包括操作人、操作类型、时间、相关文件 ID。管理员可以在前端查看日志列表。

### 4. 统计分析

用 Spark 对 HBase 中的数据进行批量分析，生成以下统计结果并写回 HBase：
- 各用户上传文件数量
- 各类型文件（pdf/jpg/txt 等）数量分布
- 每日上传量趋势
- 存储空间占用统计
- 用户相似度矩阵

同时后端也提供实时统计接口，直接从 HBase 读取数据计算，不依赖 Spark 作业的结果。

### 5. 数据可视化 Dashboard

前端用 ECharts 做一个数据看板页面，至少包含：
- 概览卡片：总文件数、总存储大小、总下载次数、用户数
- 柱状图：各用户文件数量
- 饼图/环形图：文件类型分布
- 折线图（带面积渐变）：近 7 天上传趋势

### 6. AI 智能功能

**文件摘要与自动标签：**
- 上传文本类文件时，自动调用 LLM API 生成中文摘要和分类标签
- 也支持手动触发重新生成
- 摘要和标签存入 HBase 元数据，前端可以查看

**相关文件之间的链条：**

- 将内容上有联系的文件进行关联，就比如用户在自己的主界面查看所存的文件时，有默认的浏览模式，还可以切换为关系网（图谱）模式，在关系网模式下，用户可以看到各文件之间的网状结构，方便在查看一个文件的时候，能直接看到所有跟这个文件存在联系的文件

### 7. 前端界面

至少需要这些页面/视图：
- **登录注册页**：登录/注册切换，Token 缓存到 localStorage，刷新页面自动恢复登录
- **我的文件页**：文件列表表格 + 搜索筛选 + 上传按钮 + 下载/删除/AI摘要操作按钮 + 分页 + AI 摘要弹窗
- **数据看板页**：上述统计图表
- **智能推荐页**：三种推荐模式切换 + 推荐文件列表
- **操作日志页**（仅管理员可见）：日志表格
- 左侧固定侧边栏导航，底部显示用户信息和退出按钮
- 全局 Toast 通知（成功/失败/信息）
